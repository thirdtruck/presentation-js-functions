Defining Functions

The function keyword

function myFunc (withArg) {
  console.log("This function was called, and with " + withArg + "!");
}
The "function" keyword tells the compiler that we're defining a function. The name of the function, here "myFunc." This part is optional, but we'll visit that in more detail soon. The arguments to the function, just like the arguments you passed to the methods earlier. The code of the function itself, wrapped in curly brackets.

Methods vs. Functions

The War of the Words

Well then, what does it mean for a method to "be called with a subject"? That requires some context, and a relevation: In JavaScript, methods are functions! (Big caveat: The distinction between methods and functions is stronger in some other languages.)

Creating Methods

anObject.itsMethod = function () {
  console.log("Called a method on an object.");
};
And here's how they're created: by assigning them to properties on objects. Here's the thing: This is more than just a nice way of managing your code (as opposed to defining all of your methods at the top level).

"this" is What You Wanted

anObject.aProperty = "this is a property";
anObject.aMethod = function () {
  console.log("The property aProperty is set to: " + this.aProperty");
};
anObject.aMethod()  // "This property aProperty is set to: this is a property"
          

            "What a minute," you're saying. "What's does 'this' mean, and why are you using it when you still haven't explained what it means for a method to be called with a subject?"

"this" is What it Means

anObject.aMethod = function () {
  console.log(anObject === this); // true
}
"this" is the implicit subject! Using "this," we don't have to name the subject explicitly. This will become very important soon.

Functions and "this"

function aFunc() {
  console.log(aFunc === this); // false
  console.log(window === this); // true (in a browser)
  console.log(root === this); // true (in node.js)
}
So what does that mean for functions? In JavaScript (in general) (unlike some other languages), there's always a "this". Browsers default to the "window" object and node.js defaults to "root". And, since functions in JavaScript technically have a subject, they're indistinguishable from methods in practice. (And that's even before we get into the more advanced things we can do with functions.)
